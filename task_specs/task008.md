タスク8の変換は、**水色の 2×2 正方形を固定したまま、赤い図形だけを水色の正方形へ向かって直線的にスライドさせ、接触する直前／接触する位置で止める**、というルールです。入力・出力のサイズは同じで、赤図形の形や向きは変わりません。

## ルールの言語化

盤面には主に以下の2物体があります。

* `8`: 水色の 2×2 正方形
* `2`: 赤い任意形状の図形
* `0`: 背景

変換では、**水色の正方形は一切動かさない**。
赤い図形だけを、**水色の正方形に向かって上下左右のどれか1方向へ平行移動**します。

移動方向は、赤と水色の相対位置で決まります。

* 赤が水色の上にある → 赤を下へ動かす
* 赤が水色の下にある → 赤を上へ動かす
* 赤が水色の左にある → 赤を右へ動かす
* 赤が水色の右にある → 赤を左へ動かす

そして、赤い図形は、**水色の正方形に1マス隣接する位置まで移動**します。
つまり、赤と水色の間にある空白の行または列を詰めるだけです。

## 重要な点

赤い図形は、回転・反転・変形しません。
赤いセル同士の相対位置は完全に保持されます。

水色の 2×2 正方形も、位置・形・色すべてそのままです。

変わるのは、赤い図形の位置だけです。

## 実装向けの判定方法

まず、赤図形のバウンディングボックスと、水色正方形のバウンディングボックスを取ります。

赤と水色の**列範囲が重なっている**場合は、縦方向に動かします。

* 赤が水色より上なら、赤の下端が水色の上端の1行上に来るまで下げる
* 赤が水色より下なら、赤の上端が水色の下端の1行下に来るまで上げる

赤と水色の**行範囲が重なっている**場合は、横方向に動かします。

* 赤が水色より左なら、赤の右端が水色の左端の1列左に来るまで右へ動かす
* 赤が水色より右なら、赤の左端が水色の右端の1列右に来るまで左へ動かす

## 各例での説明

train[0] では、赤図形が水色の上側にあり、列方向では接触可能な位置に重なっています。
そのため赤図形を下へスライドさせ、水色のすぐ上に接する位置まで移動します。

train[1] では、赤図形が水色の左側にあり、行方向では接触可能な位置に重なっています。
そのため赤図形を右へスライドさせ、水色のすぐ左に接する位置まで移動します。

train[2] では、赤図形が水色の下側にあります。
そのため赤図形を上へスライドさせ、水色のすぐ下に接する位置まで移動します。

test[0] では、赤図形が水色の右側にあります。
そのため赤図形を左へ2列スライドさせ、水色のすぐ右に接する位置に置きます。

## 擬似コード

```python
out = all_zero_grid_same_size_as_input

blue = cells_with_value_8
red = cells_with_value_2

blue_bbox = bbox(blue)
red_bbox = bbox(red)

if red_bbox.column_range overlaps blue_bbox.column_range:
    # 縦方向に移動
    if red_bbox.bottom < blue_bbox.top:
        dr = blue_bbox.top - red_bbox.bottom - 1
    else:
        dr = blue_bbox.bottom - red_bbox.top + 1
    dc = 0
else:
    # 横方向に移動
    if red_bbox.right < blue_bbox.left:
        dc = blue_bbox.left - red_bbox.right - 1
    else:
        dc = blue_bbox.right - red_bbox.left + 1
    dr = 0

place blue cells unchanged in out
place each red cell at (r + dr, c + dc) in out
```

一言でまとめると、**「水色の2×2ブロックを固定障害物として、赤い図形をその方向へまっすぐ滑らせ、隙間を詰めて接触させる」**タスクです。
