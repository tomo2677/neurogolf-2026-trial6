## task037 の変換ロジック

task037 は **10×10 グリッド**で、各入力には背景 `0` と複数の色付きセルがある。各色は基本的に **同じ色が2個ずつ**置かれており、その2点は必ず斜め45度方向に並んでいる。出力では、**同じ色の2点を、その色で斜め直線として結ぶ**。train は3例、test は1例の構成。

---

## ルール

1. `0` は背景なので処理対象外。
2. 非ゼロ色ごとに、その色のセル位置を2つ探す。
3. 同じ色の2点を結ぶ斜め線分を、その色で塗る。
4. 塗るのは **2点の間だけ**。盤面端まで延長しない。
5. 斜め方向は2種類のみ。

   * 左上 ↔ 右下：行と列が同じ向きに増減する
   * 右上 ↔ 左下：行と列が逆向きに増減する
6. 元の色付きセルはそのまま残り、間の空白セルだけが同じ色で補完される。
7. 背景の `0` は、どの線分にも含まれなければ `0` のまま。

train[0] では、例えば色 `2` が `(0,2)` と `(2,0)` にあり、出力では `(0,2), (1,1), (2,0)` が `2` になる。同様に色 `6` と `4` も、それぞれ同色2点の間が斜めに補完されている。
test[0] でも、色 `3`, `9`, `7`, `6`, `4` の各ペアについて、同じ処理で斜め線分が生成されている。

---

## 座標ベースでの説明

座標は `(row, col)`、0-indexed とする。

ある色 `color` の2点を

```text
(r1, c1), (r2, c2)
```

とする。

このタスクでは必ず

```text
abs(r2 - r1) == abs(c2 - c1)
```

になっている。つまり2点は斜め45度上にある。

差分方向を

```text
dr = sign(r2 - r1)
dc = sign(c2 - c1)
```

とする。

すると、出力で塗るセルは

```text
(r1 + k * dr, c1 + k * dc)
```

で、

```text
k = 0, 1, 2, ..., abs(r2 - r1)
```

まで。

そこにすべて `color` を入れる。

## 実装用擬似コード

```python
def solve(grid):
    H = len(grid)
    W = len(grid[0])

    # 元の点を保持するため copy から開始
    out = [row[:] for row in grid]

    colors = sorted(set(v for row in grid for v in row if v != 0))

    for color in colors:
        pts = []
        for r in range(H):
            for c in range(W):
                if grid[r][c] == color:
                    pts.append((r, c))

        # このタスクでは各色2点
        if len(pts) != 2:
            continue

        (r1, c1), (r2, c2) = pts

        dr_raw = r2 - r1
        dc_raw = c2 - c1

        # 斜め45度でない場合は何もしない
        # task037 では基本的にここは必ず真
        if abs(dr_raw) != abs(dc_raw):
            continue

        step_r = 0 if dr_raw == 0 else (1 if dr_raw > 0 else -1)
        step_c = 0 if dc_raw == 0 else (1 if dc_raw > 0 else -1)

        steps = abs(dr_raw)

        for k in range(steps + 1):
            r = r1 + step_r * k
            c = c1 + step_c * k
            out[r][c] = color

    return out
```

## test[0] に対する具体的な変換

入力の色ペアは以下。

```text
3: (0,6) と (3,3)
9: (0,9) と (3,6)
7: (1,0) と (7,6)
6: (4,0) と (7,3)
4: (5,9) と (9,5)
```

それぞれを斜めに結ぶ。

```text
3: (0,6), (1,5), (2,4), (3,3)
9: (0,9), (1,8), (2,7), (3,6)
7: (1,0), (2,1), (3,2), (4,3), (5,4), (6,5), (7,6)
6: (4,0), (5,1), (6,2), (7,3)
4: (5,9), (6,8), (7,7), (8,6), (9,5)
```

したがって test[0] の出力は次の通り。

```python
[
  [0,0,0,0,0,0,3,0,0,9],
  [7,0,0,0,0,3,0,0,9,0],
  [0,7,0,0,3,0,0,9,0,0],
  [0,0,7,3,0,0,9,0,0,0],
  [6,0,0,7,0,0,0,0,0,0],
  [0,6,0,0,7,0,0,0,0,4],
  [0,0,6,0,0,7,0,0,4,0],
  [0,0,0,6,0,0,7,4,0,0],
  [0,0,0,0,0,0,4,0,0,0],
  [0,0,0,0,0,4,0,0,0,0],
]
```

## ONNX 実装上の要点

このタスクは学習不要の決定的ルールでよい。

* 入力サイズは `10×10` 固定として扱ってよい。
* 色は `1..9` を走査すればよい。
* 各色について位置を2つ検出する。
* `abs(dr) == abs(dc)` のときだけ、2点間の斜め線分を埋める。
* 出力は入力コピーから始めると安全。
* 交差や競合はこのタスク例では発生していないため、単純に線分セルへその色を書けばよい。
