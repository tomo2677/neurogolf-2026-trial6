## task027 変換ロジック

このタスクは **10x10 グリッド**で、入力は `0=黒背景`, `1=青オブジェクト`、出力では元の青を残したまま、足りない対称位置を `2=赤` で補います。train は 3 件、test は 1 件です。

本質は **青オブジェクトを180度回転対称に補完する**ことです。
赤セルは「既存の青セルを正しい中心で180度回転したとき、本来青があるべきなのに入力では黒だった場所」です。

---

## 重要な見方

これは単なる左右反転ではありません。
**上下左右を同時に反転する180度回転**です。

train[1] や train[2] は上下方向にも対称っぽいので左右反転に見えますが、train[0] と test[0] では、赤の位置は **180度回転**として見ると完全に説明できます。

---

## 回転中心の決め方

グリッド座標は 0-indexed で、行 `r=0..9`, 列 `c=0..9` とします。

このタスクでは回転中心候補は実質的に次の2つです。

### 候補A: 中心 `(4.5, 4.5)`

対応セルは：

```text
(r, c) -> (9 - r, 9 - c)
```

これは普通の10x10全体の180度回転です。

### 候補B: 中心 `(5, 5)`

対応セルは：

```text
(r, c) -> (10 - r, 10 - c)
```

この場合、対応先が `0..9` の範囲外になるセルは無視します。

---

## 中心推定ルール

2つの候補を試し、**既存の青セル同士が最も多く重なる中心**を採用します。

具体的には、各候補 `S` を `9` または `10` として、

```text
対応先 = (S - r, S - c)
```

を計算します。

その対応先がグリッド内で、かつ入力でも青 `1` なら「一致」と数えます。
一致数が多い方の `S` を採用します。

train/test では以下になります。

```text
train[0] : S = 9
train[1] : S = 10
train[2] : S = 10
test[0]  : S = 9
```

---

## 出力生成手順

1. 出力を入力のコピーで初期化する。
2. 回転中心 `S` を `9` または `10` から選ぶ。
3. 入力中のすべての青セル `(r, c)` について、対応先 `(rr, cc) = (S-r, S-c)` を計算する。
4. `(rr, cc)` がグリッド内で、かつ入力では黒 `0` なら、出力のそのセルを赤 `2` にする。
5. すでに青 `1` の場所はそのまま `1` のままにする。
6. その他の黒は `0` のまま。

つまり、出力は：

```text
元の青オブジェクト + 180度回転で不足しているセルを赤で追加
```

です。

---

## 疑似コード

```python
def solve(grid):
    H, W = 10, 10
    blue = [[grid[r][c] == 1 for c in range(W)] for r in range(H)]

    def score(S):
        cnt = 0
        for r in range(H):
            for c in range(W):
                if not blue[r][c]:
                    continue
                rr = S - r
                cc = S - c
                if 0 <= rr < H and 0 <= cc < W and blue[rr][cc]:
                    cnt += 1
        return cnt

    # 中心候補は S=9 と S=10
    S = 9 if score(9) >= score(10) else 10

    out = [row[:] for row in grid]

    for r in range(H):
        for c in range(W):
            if grid[r][c] != 1:
                continue

            rr = S - r
            cc = S - c

            if 0 <= rr < H and 0 <= cc < W:
                if grid[rr][cc] == 0:
                    out[rr][cc] = 2

    return out
```

## ONNX向けの小さい実装イメージ

入力を `B = (x == 1)` の binary mask とします。

```text
rot9  = B[::-1, ::-1]
rot10 = 0で初期化し、rot10[1:10, 1:10] = B[1:10, 1:10][::-1, ::-1]
```

スコア：

```text
score9  = sum(B & rot9)
score10 = sum(B & rot10)
```

採用回転：

```text
R = rot9 if score9 >= score10 else rot10
```

赤マスク：

```text
red = R & ~B
```

出力：

```text
out = x
out[red] = 2
```

または数式的には、入力が `0/1` のみなら：

```text
out = x + 2 * red
```

`red` は元の青と重ならないので、青が `3` になることはありません。

test[0] では中心は `S=9`、つまり `(r,c)->(9-r,9-c)` です。

例：

```text

入力の青 (4,9) -> 赤 (5,0)
```
