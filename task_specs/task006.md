タスク6の変換ロジックは **左右の3x3領域の重なり抽出** です。入力は 3x7 で、中央の灰色列 `5` が区切りになっており、左側3列と右側3列がそれぞれ 3x3 の図形です。実際のJSONでも、各入力は `3x7`、出力は `3x3` になっています。

## ルール

入力を中央の灰色列で左右に分ける。

```text
左3x3 | 灰色列 | 右3x3
```

出力は 3x3。

各マスについて、**左側の同じ位置と右側の同じ位置がどちらも青 `1` のときだけ、出力で赤 `2` にする**。

それ以外のマスは黒 `0` にする。

つまり、左3x3と右3x3の **AND演算** です。

```text
output[r][c] = 2  if left[r][c] == 1 and right[r][c] == 1
output[r][c] = 0  otherwise
```

## 例

train[0] では、左と右の両方が青になっている位置は中央だけなので、出力も中央だけ赤になります。

```text
左:
1 0 0
0 1 0
1 0 0

右:
0 1 0
1 1 1
0 0 0

重なり:
0 0 0
0 2 0
0 0 0
```

train[1] では、重なっている位置が上中央・中央右・下中央なので、そこだけ赤になります。

```text
左:
1 1 0
0 0 1
1 1 0

右:
0 1 0
1 1 1
0 1 0

重なり:
0 2 0
0 0 2
0 2 0
```

train[2] でも同様に、左と右の両方が `1` の位置だけが `2` になります。

```text
左:
0 0 1
1 1 0
0 1 1

右:
0 0 0
1 0 1
1 0 1

重なり:
0 0 0
2 0 0
0 0 2
```

## test[0] への適用

```text
左:
1 0 1
0 1 0
1 0 1

右:
1 0 1
1 0 1
0 1 0
```

両方が青 `1` の位置は、上段左と上段右だけです。

したがって出力は次のようになります。

```text
2 0 2
0 0 0
0 0 0
```

## 実装向けの簡潔な表現

中央の `5` の列を無視し、左3列 `input[:, 0:3]` と右3列 `input[:, 4:7]` を比較する。
同じ座標で両方が `1` のセルを `2` にし、それ以外を `0` にする。
