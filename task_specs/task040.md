# task040 の変換ロジック

根拠データでは、全例が `10x10` で、train が 3 件、test が 1 件です。

## ルール概要

入力には必ず次の3種類の要素がある。

* 背景: `0`
* 塗り替え対象マーカー: `3`
* 盤面の両端にある2色のガイド線

  * 左右の端列にある場合
  * または上下の端行にある場合

出力では、`3` の位置は動かさず、その `3` を「近い方の端の色」に塗り替える。

背景 `0` と、端にあるガイド線はそのまま残す。

## 判定方法

### 1. 左右にガイド線がある場合

例: train[0] や test[0]

* 左端列が同じ色で埋まっている
* 右端列が別の色で埋まっている
* 各 `3` について、列位置を見る
* 左端に近ければ左端の色
* 右端に近ければ右端の色

`10x10` なので、列番号を `0` 始まりで見ると次の分割になる。

* `col <= 4` なら左端の色
* `col >= 5` なら右端の色

test[0] では左端色が `5`、右端色が `4` なので、左半分の `3` は `5`、右半分の `3` は `4` に変わる。

### 2. 上下にガイド線がある場合

例: train[1], train[2]

* 上端行が同じ色で埋まっている
* 下端行が別の色で埋まっている
* 各 `3` について、行位置を見る
* 上端に近ければ上端の色
* 下端に近ければ下端の色

`10x10` なので、行番号を `0` 始まりで見ると次の分割になる。

* `row <= 4` なら上端の色
* `row >= 5` なら下端の色

train[1] では上端色 `4`、下端色 `7` なので、上半分の `3` は `4`、下半分の `3` は `7` になる。train[2] では上端色 `8`、下端色 `9` なので、上半分の `3` は `8`、下半分の `3` は `9` になる。

## 実装用の手順

```text
入力 grid をコピーして output を作る。

H = 10
W = 10

もし row 0 と row H-1 が、それぞれ非ゼロの同一色で横一列に埋まっているなら:
    top_color = grid[0][0]
    bottom_color = grid[H-1][0]

    全セルを走査し、grid[r][c] == 3 のセルについて:
        if r <= 4:
            output[r][c] = top_color
        else:
            output[r][c] = bottom_color

そうでなく、col 0 と col W-1 が、それぞれ非ゼロの同一色で縦一列に埋まっているなら:
    left_color = grid[0][0]
    right_color = grid[0][W-1]

    全セルを走査し、grid[r][c] == 3 のセルについて:
        if c <= 4:
            output[r][c] = left_color
        else:
            output[r][c] = right_color

それ以外のセルは変更しない。
```

## 汎化した条件式

`10x10` 固定なら `4/5` 境界で十分だが、少し汎化するなら次のように書ける。

```text
左右ガイドの場合:
    if c * 2 <= W - 1:
        左端色
    else:
        右端色

上下ガイドの場合:
    if r * 2 <= H - 1:
        上端色
    else:
        下端色
```

## 一言でいうと

`3` を、上下または左右どちらかにある2本の色ガイドのうち、近い側の色に置換するタスク。位置は一切動かさない。
