タスク2のロジックは、**緑色の線で囲まれた黒い領域を黄色で塗りつぶす**処理です。
色としては、`0=黒背景`, `3=緑の壁・輪郭`, `4=黄色の塗りつぶし`です。

## 変換ルール

入力の緑セル `3` はそのまま残します。
黒セル `0` のうち、**グリッドの外側・端から上下左右にたどって到達できない黒セル**を、黄色 `4` に変えます。

つまり、緑セルを「壁」と見なして、外側から侵入できない黒い穴だけを塗ります。

重要なのは、接続判定は **上下左右の4方向のみ** です。
斜め方向でつながっていても、外側とつながっているとは見なしません。

## 手順として言語化すると

1. 出力はまず入力をコピーする。
2. 緑セル `3` は壁として扱う。
3. グリッドの外周にある黒セル `0` から、上下左右に flood fill して「外側とつながっている黒領域」を見つける。
4. flood fill で到達できた黒セルは、そのまま `0` にする。
5. flood fill で到達できなかった黒セルは、緑の壁に囲まれた内部なので `4` に変える。
6. 緑セル `3` はすべて元のまま維持する。

## 例で見ると

train[0] では、緑が斜めに並んだ小さい形の中に、上下左右をすべて緑で囲まれた黒セルが2個あります。
その2個だけが黄色になります。

train[2] では、中央に縦長の緑の輪郭があります。
その輪郭の内側の黒セルがまとめて黄色になります。
右側の小さな菱形っぽい部分の内部も、外側から4方向で入れないので黄色になります。

train[4] や test[0] では、複数の囲いがあります。
大きな長方形、細い輪郭、小さな菱形など、形の種類に関係なく、**外側から到達不能な黒領域**がすべて黄色になります。
孤立した緑点や、閉じていない線は壁にはなりますが、黒領域を完全に囲んでいなければ黄色は発生しません。

## 実装向けの擬似コード

```python
out = copy(input)

# visited: 外側から到達できる黒セル
visited = all False

# 外周の 0 から BFS / DFS
queue = []
for each border cell:
    if input[cell] == 0:
        visited[cell] = True
        queue.append(cell)

while queue:
    r, c = queue.pop()
    for nr, nc in up/down/left/right neighbors:
        if inside grid and input[nr][nc] == 0 and not visited[nr][nc]:
            visited[nr][nc] = True
            queue.append((nr, nc))

# 到達できなかった 0 を 4 にする
for each cell:
    if input[cell] == 0 and not visited[cell]:
        out[cell] = 4
    else:
        out[cell] = input[cell]
```

## まとめ

このタスクは、**緑の線を壁として扱い、黒背景のうち外部に連結していない閉領域を黄色で塗る**問題です。
ONNX化するなら、ロジックの本質は「4近傍 flood fill による外部連結判定」です。
