## task016 の変換ロジック

参照データ: `task016.json` と `manifest.json` に基づく。 

### 観察

入力も出力も常に **3×3** のグリッドです。
各列は縦方向に同じ色で塗られており、出力でも列構造や位置は一切変わりません。

つまり、このタスクでは図形の移動・回転・反転・拡大縮小はなく、**各セルの色だけを別の色に置き換える**処理です。

### 変換規則

各ピクセルの色を、以下の固定ペアで相互変換します。

| 入力色 | 出力色 |
| --: | --: |
|   1 |   5 |
|   5 |   1 |
|   2 |   6 |
|   6 |   2 |
|   3 |   4 |
|   4 |   3 |
|   8 |   9 |
|   9 |   8 |

これは、次のような色ペアの入れ替えとして表せます。

```text
1 ↔ 5
2 ↔ 6
3 ↔ 4
8 ↔ 9
```

### 重要な点

* グリッドのサイズはそのまま保持する。
* 各セルの位置もそのまま保持する。
* 行・列の並べ替えは行わない。
* 入力の各色を、対応するペアの反対側の色に置き換える。
* このタスクでは `0` と `7` は出現していないため、実装上は未定義色としてそのまま残す方針でよい。

### 実装向け疑似コード

```python
def solve(grid):
    mapping = {
        1: 5,
        5: 1,
        2: 6,
        6: 2,
        3: 4,
        4: 3,
        8: 9,
        9: 8,
    }

    h = len(grid)
    w = len(grid[0])

    out = [[0 for _ in range(w)] for _ in range(h)]

    for r in range(h):
        for c in range(w):
            color = grid[r][c]
            out[r][c] = mapping.get(color, color)

    return out
```

### ONNX 化を意識した最小実装方針

これは完全な **要素ごとの色変換** なので、座標情報や周囲ピクセルを見る必要はありません。

最も単純には、色番号をインデックスにした LUT を使えます。

```text
lut[0] = 0
lut[1] = 5
lut[2] = 6
lut[3] = 4
lut[4] = 3
lut[5] = 1
lut[6] = 2
lut[7] = 7
lut[8] = 9
lut[9] = 8
```

入力グリッド `x` の各要素に対して、

```text
output[r][c] = lut[x[r][c]]
```

を適用すればよいです。
